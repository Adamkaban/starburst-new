# Frontmatter Rules — All Page Types

## FIRST: identify page type (see CLAUDE.md), then use the correct template below.

---

## TYPE 1 — Content pages (.md files in src/content/pages/)
Used for: features, how-to-play, rtp-volatility, bonuses, strategy, mobile, casinos, faq, review

```yaml
---
title: ""
description: ""
publishedDate: "YYYY-MM-DD"
lastUpdated: "YYYY-MM-DD"
ogImage: "/images/og/[slug].webp"
ogImageAlt: ""
author: "johan-van-der-merwe"
noindex: false
schema: "article"
---
```

Field rules:
- title: ≤44 chars WITHOUT spaces | pattern: "[Topic] – Starburst Slots ZA"
  ✅ "Best Starburst Casinos SA 2026 – Starburst Slots ZA"
  ❌ "Best Starburst Casinos South Africa 2026"  (no brand suffix)
- description: ≤220 chars | target 140–160 | must contain primary keyword
  Do NOT put promo codes in description — codes change, description is cached
- publishedDate: set once, never change
- lastUpdated: update EVERY time file is edited
- ogImage: always /images/og/[slug].webp — must exist in public/images/og/
- ogImageAlt: describe the image, not the page topic
- author: always exactly "johan-van-der-merwe" — no variations
- noindex: false for all Type 1 pages
- schema: "article" for all Type 1 pages including casinos
  Exception: casinos.md also gets ItemList schema — handled by [slug].astro
  detecting slug === "casinos" and adding second JSON-LD block

Canonical is auto-generated by [slug].astro:
  https://starburstslots.co.za/ + slug + /
Do NOT add canonical field to Type 1 frontmatter.

### Casinos page special rules
- schema: "article" in frontmatter (Article JSON-LD)
- [slug].astro detects slug === "casinos" → adds ItemList JSON-LD automatically
- Promo codes in body text must match PROMO_CODE in src/data/casinos.ts
- Internal links must use correct routes (see content.md internal links section)
- No raw affiliate URLs in .md — use text only, links rendered by template

## TYPE 3 — Unique pages (.astro files, fully custom)
Used for: index.astro, about.astro, responsible-gambling.astro,
          privacy-policy.astro, terms-of-use.astro

```astro
---
const meta = {
  title:       "",
  description: "",
  canonical:   "https://starburstslots.co.za/",
  noindex:     false,
  ogImage:     "/images/og/[slug].webp",
  ogImageAlt:  "",
  schema:      "",
}
---
```

Schema values by page:
- index.astro                → schema: "homepage"  WebSite + Organization + Review
- about.astro                → schema: "person"    Person (Author) JSON-LD
- responsible-gambling.astro → schema: "webpage"
- privacy-policy.astro       → schema: "webpage"   noindex: true
- terms-of-use.astro         → schema: "webpage"   noindex: true

---

## Validation checklist (run before saving any file)

Title — count chars WITHOUT spaces:
  Remove all spaces from title string, count remaining characters. Must be ≤44.

Description — count total chars including spaces. Must be ≤220. Target 140–160.
  Never include promo codes in description.

ogImage — file must exist at public/images/og/[slug].webp, dimensions 1200×630px.

Canonical (Type 2 and 3 only):
  Must start with https://
  Must use starburstslots.co.za (non-www)
  Must end with trailing slash /

noindex:
  true only → privacy-policy, terms-of-use
  All other pages: false (explicit, never omitted)
